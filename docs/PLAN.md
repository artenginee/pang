# PLAN.md — Pang 게임 Phase별 개발 계획

> 각 Phase는 독립적으로 실행 가능한 결과물을 목표로 한다.
> 고객님께서 `npm run dev` 후 브라우저에서 직접 확인하고 피드백을 주시면 다음 Phase로 진행한다.

---

## Phase 1 — 메인 화면

### 개발 목표
- 게임 타이틀이 표시되는 메인 화면 구성
- 세 가지 메뉴 버튼 표시: **게임 시작**, **게임 방법**, **게임 종료**
- 버튼 클릭/키보드 선택 동작 (실제 기능은 다음 Phase에서 연결)

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 타이틀 표시 | "PANG" 타이틀이 화면 상단에 보인다 |
| 버튼 3개 | 게임 시작 / 게임 방법 / 게임 종료 버튼이 보인다 |
| 버튼 hover | 마우스를 올리면 버튼이 강조된다 |
| 게임 종료 버튼 | 클릭 시 탭이 닫히거나 "종료되었습니다" 메시지가 표시된다 |

---

## Phase 2 — 게임 방법 화면

### 개발 목표
- **게임 방법** 버튼 클릭 시 조작법 안내 화면으로 이동
- 키보드 조작 안내 (`←` `→` 이동, `Space` 발사) 표시
- **돌아가기** 버튼으로 메인 화면 복귀

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 화면 이동 | 메인에서 "게임 방법" 클릭 시 안내 화면이 뜬다 |
| 조작법 내용 | 이동키, 발사키 설명이 명확하게 표시된다 |
| 메인 복귀 | "돌아가기" 버튼 클릭 시 메인 화면으로 돌아간다 |

---

## Phase 3 — 게임 캔버스 + 기본 레이아웃

### 개발 목표
- **게임 시작** 버튼 클릭 시 게임 화면으로 전환
- 고정 크기의 게임 캔버스 영역 표시 (배경색으로 구분)
- 상단 HUD 영역 (점수, 목숨, 타이머 자리) 레이아웃만 구성

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 화면 전환 | 메인에서 "게임 시작" 클릭 시 게임 화면으로 바뀐다 |
| 게임 영역 | 게임이 펼쳐질 사각형 영역이 화면에 보인다 |
| HUD 영역 | 상단에 점수 / 목숨 / 타이머 텍스트 자리가 표시된다 |

---

## Phase 4 — 플레이어 렌더링 + 이동

### 개발 목표
- 플레이어 캐릭터가 게임 영역 하단 중앙에 표시
- `←` `→` 키로 좌우 이동
- 게임 영역 경계에서 이동 멈춤 (벽 충돌)

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 캐릭터 표시 | 플레이어가 화면 하단 중앙에 보인다 |
| 좌우 이동 | `←` `→` 키를 누르는 동안 캐릭터가 이동한다 |
| 벽 막힘 | 게임 영역 끝에서 더 이상 나가지 않는다 |

---

## Phase 5 — 공(버블) 등장 + 물리 동작

### 개발 목표
- Large 공 2개가 화면에 등장
- 포물선 궤적(중력 적용)으로 위아래로 튀어 오름
- 좌우 벽과 바닥에서 반사

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 공 등장 | 게임 시작 시 Large 공 2개가 화면에 나타난다 |
| 포물선 궤적 | 공이 위로 올라갔다 아래로 내려오는 자연스러운 움직임을 보인다 |
| 벽/바닥 반사 | 벽에 닿으면 좌우가 반전되고, 바닥에 닿으면 위로 튀어 오른다 |

---

## Phase 6 — 공 크기별 속도 / 높이 차이

### 개발 목표
- Large / Medium / Small 공의 크기를 시각적으로 구분
- 크기별로 이동 속도와 최대 높이를 다르게 설정
- (아직 분열은 없음. 크기별 공을 화면에 동시에 띄워 차이 확인)

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 크기 구분 | Large / Medium / Small 공이 크기별로 다르게 보인다 |
| 속도 차이 | Small이 가장 빠르고, Large가 가장 느리다 |
| 높이 차이 | Large가 가장 높이 튀고, Small이 가장 낮게 튄다 |

---

## Phase 7 — 작살(Harpoon) 발사

### 개발 목표
- `Space` 키로 플레이어 위 방향으로 작살 발사
- 작살이 천장에 닿으면 사라지고 재발사 가능
- 작살이 살아있는 동안 재발사 불가

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 작살 발사 | `Space`를 누르면 캐릭터 위로 선이 뻗어 올라간다 |
| 천장 소멸 | 선이 천장에 닿으면 사라진다 |
| 연사 제한 | 선이 있는 동안 `Space`를 눌러도 추가 발사가 안 된다 |

---

## Phase 8 — 작살과 공 충돌 + 분열

### 개발 목표
- 작살이 공에 닿으면 공 제거 및 분열
  - Large → Medium × 2
  - Medium → Small × 2
  - Small → 완전 소멸
- 분열된 공은 좌우로 퍼지며 독립적으로 물리 동작

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| Large 분열 | 작살이 Large에 닿으면 Medium 2개로 나뉜다 |
| Medium 분열 | 작살이 Medium에 닿으면 Small 2개로 나뉜다 |
| Small 소멸 | 작살이 Small에 닿으면 완전히 사라진다 |
| 분열 후 물리 | 분열된 공이 각각 독립적으로 튀어다닌다 |

---

## Phase 9 — 플레이어-공 충돌 + 사망 처리

### 개발 목표
- 플레이어와 공이 겹치면 사망 판정
- 사망 시 잠시 무적 후 제자리 복귀
- 목숨 1 차감 및 HUD에 반영

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 사망 판정 | 공이 플레이어에 닿으면 반응(깜빡임 등)이 일어난다 |
| 목숨 차감 | HUD의 목숨 수가 1 줄어든다 |
| 복귀 | 잠시 후 플레이어가 제자리에서 다시 조작 가능해진다 |

---

## Phase 10 — 목숨 소진 + 게임 오버 화면

### 개발 목표
- 목숨이 0이 되면 게임 오버 화면으로 전환
- 게임 오버 화면에서 **다시 시작** / **메인으로** 버튼 제공

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 게임 오버 전환 | 목숨이 0이 되면 게임 오버 화면이 뜬다 |
| 다시 시작 | 버튼 클릭 시 목숨 3개로 Mission 1이 다시 시작된다 |
| 메인으로 | 버튼 클릭 시 메인 화면으로 돌아간다 |

---

## Phase 11 — 제한 시간 + 타임오버

### 개발 목표
- HUD에 60초 카운트다운 타이머 표시
- 0초 도달 시 목숨 1 차감 후 스테이지 재시작
- 목숨이 없으면 게임 오버로 연결

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 타이머 표시 | 게임 중 상단에 남은 시간이 숫자로 줄어든다 |
| 타임오버 | 0초가 되면 목숨이 차감되고 스테이지가 재시작된다 |
| 연동 | 타임오버로 목숨이 0이 되면 게임 오버 화면이 뜬다 |

---

## Phase 12 — 스테이지 클리어 화면

### 개발 목표
- 화면의 모든 공이 소멸되면 클리어 화면으로 전환
- **다시 시작** / **메인으로** 버튼 제공

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 클리어 전환 | 모든 공 제거 시 클리어 화면이 뜬다 |
| 다시 시작 | 버튼 클릭 시 Mission 1을 처음부터 다시 시작한다 |
| 메인으로 | 버튼 클릭 시 메인 화면으로 돌아간다 |

---

## Phase 13 — 점수 시스템

### 개발 목표
- 공 크기별 점수 부여 (Small > Medium > Large)
- HUD에 현재 점수 실시간 표시
- 스테이지 클리어 시 잔여 시간 × 보너스 점수 계산 및 표시
- 세션 내 최고 점수(HI-SCORE) 메인 화면에 표시

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 점수 증가 | 공을 제거할 때마다 HUD 점수가 올라간다 |
| 크기별 차등 | Small을 제거할 때 가장 높은 점수가 오른다 |
| 타임 보너스 | 클리어 화면에서 보너스 점수가 별도로 표시된다 |
| HI-SCORE | 메인 화면에 이번 세션 최고 점수가 유지된다 |

---

## Phase 14 — 아이템 드롭 + 획득

### 개발 목표
- 공 제거 시 확률적으로 아이템이 바닥으로 떨어짐
- 플레이어가 아이템에 닿으면 효과 적용
- 아이템 종류: 시계(공 일시정지), 방어막(1회 피격 무효)

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 아이템 드롭 | 공 제거 시 가끔 아이템이 떨어진다 |
| 시계 아이템 | 획득 시 모든 공이 잠시 멈춘다 |
| 방어막 아이템 | 획득 후 공에 한 번 닿아도 죽지 않는다 |
| 소멸 | 일정 시간 후 획득 안 한 아이템이 사라진다 |

---

## Phase 15 — 추가 무기 (Wire / Double Harpoon)

### 개발 목표
- 아이템으로 무기 교체 기능 추가
- **Wire**: 발사 즉시 뻗었다 사라지는 근접형, 연사 가능
- **Double Harpoon**: 좌우 양방향 동시 발사

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| Wire | 발사 후 빠르게 뻗었다 즉시 사라지며 연속 발사된다 |
| Double Harpoon | `Space` 한 번에 좌우로 두 줄이 동시에 뻗는다 |
| 무기 전환 | 아이템 획득 시 현재 무기가 교체되어 동작이 바뀐다 |

---

## Phase 16 — 시각 효과 + 전체 완성도

### 개발 목표
- 공 분열 / 소멸 시 폭발 이펙트
- 배경 (후지산 이미지 또는 색상 배경) 적용
- 전체 화면 전환 애니메이션 및 UI 다듬기

### 고객님 확인 포인트
| 확인 항목 | 기대 동작 |
|-----------|-----------|
| 폭발 이펙트 | 공이 소멸할 때 짧은 애니메이션이 재생된다 |
| 배경 | 게임 화면에 배경이 적용되어 있다 |
| 전체 흐름 | 메인 → 게임 → 클리어/게임오버 → 메인 전환이 자연스럽다 |
| 완성도 | 전반적인 게임 경험이 팡 게임과 유사하게 느껴진다 |
